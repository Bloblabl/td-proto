# Решения по открытым вопросам ТЗ (§14)

Утверждено владельцем продукта 17.07.2026.

| # | Вопрос | Решение |
|---|---|---|
| 1 | Phaser 3 или Unity | **Phaser 3 для прототипа**, позже порт на Unity. Следствие: ядро `src/core/` держать строго без зависимостей от Phaser и DOM — оно переносится в C# почти механически. |
| 2 | Урон базе | **«Прошёл — исчез»** (классика): монстр, прошедший полный круг, снимает 1 жизнь и удаляется. В конфиге: `lapsDamage: false` (применено). |
| 3 | Радиусы атаки юнитов | **Не добавлять** (`range_enabled` не реализуем): все юниты бьют весь трек. |
| 4 | Колода | **Выбор 5 из 7 типов перед забегом** — реализовать на этапе 4 вместе с драфтом. Требуется: 2 новых типа юнита (кандидаты: «Яд» — DoT, «Разряд» — chain lightning) и экран выбора колоды. |

## Следствия для баланса

- При `lapsDamage: false` монстр проходит каждый тайл трека ровно один раз — заграждения (этап 3)
  срабатывают по каждому монстру однократно. Их численность/длительность можно балансировать щедрее,
  чем в режиме кругов.
- Таргетинг `first` (по `travelled`) при исчезновении монстров работает корректно — пройденный путь
  монотонен в пределах одного круга.
