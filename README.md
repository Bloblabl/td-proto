# TD-Proto — прототип механик merge-TD

Все 5 этапов ТЗ: ядро (поле, трек, волны, мана, призыв), мердж с diminishing returns и in-match апгрейдом типов, заграждения, бусты, Селектор, режим «Аркада» (драфт 1-из-3 каждые 5 волн, колода 5 из 7), телеметрия по волнам и debug-инструменты. Дополнительно: режимы «Испытание дня» и Boss Rush. Стек: TypeScript strict + Phaser 3 + Vite. Решения по открытым вопросам — в `DECISIONS.md`.

## Запуск

```bash
npm install
npm run dev        # http://localhost:5173
npm run build      # tsc --noEmit + сборка в dist/ (статика для деплоя)
npm run smoke      # headless-прогон: детерминизм + механики (гейт CI)
npm run balance 200  # массовые балансные симуляции на N сидов
npm run server     # auth-сервер (Node + SQLite) на http://localhost:3000
```

## Авторизация и сохранение прогресса

Гость — прогресс в localStorage браузера. Вход в аккаунт — прогресс на сервере
(мини-БД SQLite), доступен с любого устройства.

- Сервер: `npm run server` (Express + встроенный `node:sqlite`, файл БД
  `server/data/td.db`, в git не коммитится). Пароли хешируются (bcrypt), сессия — JWT.
- В dev Vite проксирует `/api` на `localhost:3000` (см. `vite.config.ts`).
- Эндпоинты: `POST /api/register`, `POST /api/login`, `GET|PUT /api/progress`.
- На задеплоенном GitHub Pages сервера нет (только статика) — там гостевой режим.
  Для авторизации на проде сервер нужно захостить отдельно (Render/VPS);
  `JWT_SECRET` задать через переменную окружения.

Живая версия для плейтеста: **https://bloblabl.github.io/td-proto/** (исходники: https://github.com/Bloblabl/td-proto).
Деплой автоматический: пуш в `master` запускает `.github/workflows/deploy.yml` — smoke → сборка → публикация. Если smoke или проверка типов падают, билд на прод не уезжает.

Фиксированный сид: `http://localhost:5173/?seed=12345` — RNG призыва, мерджа, драфта и процедурных волн детерминирован. Колода: `&deck=gunner,volley,frost,gen,sniper` (без параметра — экран выбора 5 из 7 перед забегом; рестарты сохраняют сид и колоду).

## Управление

- **Призыв** — случайный юнит ранга 1 на случайную свободную клетку, цена растёт (+10 за призыв).
- **Мердж** — перетащить юнит на юнит того же типа и ранга: ранг +1, тип случайный. Ранг даёт ×1.6 к скорости атаки, поэтому смердженный юнит суммарно слабее двух исходных (merge efficiency 0.8) — ключевое решение «мерджить или копить».
- **Апгрейды** — in-match уровень типа: +40% к базовому урону/эффекту, 100/200/400/800 маны.
- **Заграждения** (расходники, кнопка → тап по тайлу трека): Баррикада 500 HP блокирует монстров; Шипы 30 урона/0.5 с всем на тайле, 10 с; Слоу-зона −50% на 3 тайлах, 15 с (босс −25%). Лимиты активных: 2/3/2.
- **Бусты** (кулдауны, старт заряжен на 50%): Метеор 500 AoE по точке (45 с), Перегрев +100% скорости атаки 8 с (60 с), Прилив +150 маны (90 с).
- **Селектор** (расходник) — задаёт тип юнита от следующего мерджа.
- **Формы башен**: каждый тип — свой силуэт (▲ Стрелок, ✦ Залп, ⬡ Мороз, ◆ Генератор, ◎ Снайпер, 💧 Яд, ⚡ Разряд). При перетаскивании подходящие для мерджа юниты обводятся зелёным.
- **Крит и мета-прокачка**: за забег начисляются ядра (волны + босс-волны + убийства). В меню «Прокачка башен» ядра тратятся на уровни: уровень даёт +5% урона своему типу и поднимает **общий** крит (шанс и множитель) — качать выгодно любую башню. Крит роллится раз на атаку, критовый выстрел подсвечивается жёлтым. Прогресс в localStorage.
- **Колода 5 из 18**: перед забегом выбираются 5 типов. Статусные — Яд, Разряд (цепь), **Пиромант** (Горение), **Ливень** (Мокро), **Дробитель** (Разлом), **Арканист** (детонатор), **Медуза** (окаменение). Архетипы — **Вихрь** (разгон по цели), **Пушка** (тяжёлый AoE), **Ростовик** (растёт с волнами), **Палач** (казнь ниже 15% HP), **Вампир** (урон→мана), **Мим** (джокер мерджа: сливается с любым типом своего ранга).
- **Редкости драфта**: карточки common / ◆ rare / ★ legendary (легендарные редки: +1 уровень всем типам, зарядка бустов, призывы ранга 3).
- **Элементы и реакции** (в духе Magicka): статусы на монстрах реагируют. Мокро+Мороз = заморозка (стан); Яд+Горение = детонация (взрыв AoE); Мокро+Разряд = проводимость (цепь бьёт мокрых сильнее); Мокро+Горение = пар (гасит). **Разряд** по цели со статусом разносит Яд/Горение/Мороз по цепи. **Дробитель** ломает броню (уязвимость+слоу). **Арканист** детонирует все статусы цели разом (урон ×число статусов²). **Медуза** с шансом окаменяет (стан с иммунитетом от перма-стана). DoT гибридный: плоский урон + % текущего HP от уровня башни. Заморозка/окаменение дают иммунитет к стану. Метеор бьёт % max HP, Шипы — % текущего HP.
- **Драфт («Аркада»)**: после зачистки волн 5, 10, 15… — пауза и выбор 1 из 3 карточек (пул 12: +15% урона/эффекта типу ×5, +расходники ×4, −15% кулдауна бустов, +20% маны с убийств, 3 призыва ранга 2). Стакаются, действуют до конца забега.
- **Волна ▶** — вызвать следующую волну раньше, +25 маны; после зачистки волны — пауза 2.5 с.
- **Режимы**: `?mode=daily` — «Испытание дня»: сид из даты (UTC), фиксированная колода, у всех игроков одинаковый забег; `?mode=boss` — Boss Rush: босс каждую волну (`waves_boss.json`). Кнопки перехода — в дебаг-баре.
- Дебаг-бар: скорость ×1/×2/×4, пауза, FPS + счётчик монстров, +1000 маны, +расходники, убить волну, прыжок на волну N, ввод сида, godmode, DPS-оверлей (окно 10 с по источникам), выгрузка телеметрии; в консоли доступен `td.sim`.
- Телеметрия: урон и overkill по источникам, «купленное время» баррикады и слоу-зоны, использования бустов по волнам, доля маны Генератора, поволновая статистика (мана заработана/потрачена, мерджи, распределение рангов на конец волны) — всё в JSON по кнопке «Телеметрия».

## Структура

```
public/config/balance.json   все числа баланса (хардкод в коде запрещён)
public/config/waves.json     волны 1–15 + параметры процедурной генерации
src/core/                    симуляция: без Phaser и DOM, детерминированная, тик 50 мс
  rng.ts                     mulberry32
  track.ts                   геометрия замкнутого трека (периметр кольца)
  sim.ts                     состояние, тик, API для UI (summon/merge/upgrade)
  meta.ts                    мета-прогрессия: крит, цены уровней, награда за забег
  types.ts                   типы конфигов и состояния
src/render/scene.ts          Phaser-сцена: чтение состояния sim, drag-мердж
src/render/shapes.ts         силуэты типов (Phaser-фигуры + SVG для DOM)
src/ui/hud.ts                DOM-HUD, панель апгрейдов, дебаг, экран итогов
src/ui/menu.ts               главное меню и экран прокачки
src/ui/storage.ts            localStorage (единственная точка привязки к браузеру)
src/main.ts                  загрузка конфигов, игровой цикл, Phaser bootstrap
```

Ядро (`src/core`) не импортирует Phaser — его можно гонять headless нодой для массовых симуляций баланса.
