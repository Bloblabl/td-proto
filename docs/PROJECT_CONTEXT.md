# TD-Proto — контекст проекта (handoff)

Единый самодостаточный документ для продолжения работы над проектом в новом
чате/сессии. Собран из памяти проекта, README, DECISIONS.md, дизайн-документа,
конфига и истории 16 PR. Актуально на 24.07.2026, версия 0.2.0.

## 1. Что это

Играбельный прототип мобильной **merge tower defense** (в духе Random Dice /
Rush Royale) с системой элементов в духе Magicka. Начинался как проверка механик
по ТЗ v1.0 «TD-Proto», по запросам владельца дорос до контент-игры.

- **Локальная папка:** `C:\Users\user\Documents\td`
- **Репозиторий:** https://github.com/Bloblabl/td-proto (публичный, аккаунт `Bloblabl`)
- **Живой билд:** https://bloblabl.github.io/td-proto/
- **Версия:** 0.2.0 (в меню — `v0.2.0 · <git-sha> · <дата>`)
- **Стек:** TypeScript strict + Phaser 3 + Vite. Бэкенд: Express + `node:sqlite`.
- **Рабочий язык:** русский (комментарии, коммиты, UI).

Ключевой архитектурный принцип: **ядро `src/core/` не импортирует Phaser и DOM** —
детерминированная симуляция (fixed timestep 50 мс, RNG mulberry32 от сида). Это
позволяет гонять её headless для симуляций баланса и в будущем механически
портировать на Unity/C#. Хардкод чисел баланса в коде запрещён (всё в `balance.json`).

## 3. Как запускать и разрабатывать

```bash
npm install
npm run dev         # http://localhost:5173 (Vite, HMR)
npm run build       # tsc --noEmit + vite build → dist/
npm run smoke       # headless-гейт: детерминизм + механики
npm run balance 200 # массовые балансные симуляции на N сидов
npm run server      # auth-сервер (Node --experimental-sqlite) на :3000
```

- **URL-параметры:** `?seed=12345`, `&deck=gunner,volley,frost,gen,sniper`, `?mode=daily|boss|arcade`.
- **Дебаг из консоли браузера:** `window.td.sim` / `td.controls` / `td.scene` / `td.meta`.
- **Админ-режим:** тумблер в меню (localStorage `td-proto-admin`) → дебаг-бар с читами (godmode, +мана, прыжок на волну, DPS-оверлей, выгрузка телеметрии, переключение режимов).

### Деплой и CI
- Пуш в `master` → `.github/workflows/deploy.yml`: `npm ci` → smoke → build → GitHub Pages. Падение smoke/типов не пускает на прод. Node 22.
- **Workflow сессии:** ветка от master → правки → build+smoke+симуляции → браузер-проверка → commit → PR → merge --delete-branch → дождаться deploy.
- **Грабли инструментов:** PowerShell ломает кириллицу/кавычки в `git commit -m` → всегда `git commit -F <файл>`. Скриншоты браузера таймаутят когда вкладка в фоне (rAF не тикает) → проверять через `javascript_tool` + `window.td.sim`, для боя вручную гонять `s.tick(0.05)` в цикле.

## 4. Игровые механики (всё реализовано)

**Ядро:** поле 3×5, замкнутый трек, монстры волнами по кругу, полный круг = −1 жизнь
и исчезновение (`lapsDamage:false`). Мана с убийств. Призыв юнита ранга 1 (цена
растёт). Мердж двух юнитов **того же типа и ранга** → ранг+1, тип случайный.
`rankAtkSpeedMult:1.75` (merge efficiency ~0.88) держит решение «мерджить/копить»
неоднозначным. In-match апгрейд типа (6 уровней). 3 жизни. maxRank 7.

**Мердж-нюансы:** строгое правило тип+ранг; от RNG-тупика полного поля страхует
**умный призыв** (`pickSummonType`, заполнение ≥0.6 без пары → тип подкручивается
скрыто); индикатор «⚠ ходов нет»; `canMerge()`/`hasMergeMove()` публичны. **Мим** —
джокер: сливается с любым типом своего ранга, результат берёт тип напарника.

**18 типов башен (колода 5 из 18):**
`gunner,volley,frost,gen,sniper,poison,arc,pyro,rain,crush,arcane,medusa,whirl,cannon,growth,exec,vamp,mimic`
- Базовые: Стрелок (одиночный), Залп (AoE), Мороз (слоу+уязвимость), Генератор (мана), Снайпер (по текущему max HP).
- Статусные: Яд (DoT), Разряд (цепь+разнос статусов), Пиромант (Горение), Ливень (Мокро), Дробитель (Разлом), Арканист (детонатор всех статусов), Медуза (окаменение).
- Архетипы: Вихрь (разгон по цели), Пушка (тяжёлый AoE), Ростовик (+6%/волну), Палач (казнь <15% HP), Вампир (25% урона→мана), Мим (джокер).

**Система элементов (Magicka) — статусы на монстрах реагируют:**
- Статусы: Мороз (slow + уязвимость ×1.3), Яд/Горение (DoT: плоский + %ТЕКУЩЕГО hp от уровня), Мокро (enabler), Разлом (уязвимость ×1.4 + слоу), стан (заморозка/окаменение с иммунитетом 6с).
- Реакции: Мокро+Мороз=Заморозка, Яд+Горение=Детонация(AoE), Мокро+Разряд=Проводимость, Мокро+Горение=Пар. Разряд по статусной цели **разносит Яд/Горение/Мороз** (НЕ Мокро/Заряд). Арканист детонирует все статусы цели (урон ×число_статусов²).
- Параметры реакций — `balance.json → statusFx`.

**Мета-прогрессия:** за забег «ядра» (волны+боссы+убийства); в «Прокачке башен»
уровень даёт +5% урона типу И общий крит (качать выгодно любую). Сохранение:
localStorage (гость) / сервер (залогинен).

**Заграждения:** Баррикада (блок), Шипы (8% текущего HP/тик), Слоу-зона.
**Бусты:** Метеор (25% max HP AoE), Перегрев (+100% скорости атаки), Прилив маны.
**Селектор:** тип следующего мерджа.

**Драфт (Аркада):** после волн 5/10/15… выбор 1 из 3 карточек с **редкостями**
(common / ◆ rare / ★ legendary, веса 1.0/0.5/0.18). Легендарные: +1 уровень всем
типам, зарядка бустов, призывы ранга 3.

**Режимы:** Аркада, `?mode=daily` (сид из даты UTC, фикс-колода), `?mode=boss`
(босс каждую волну). Выход из игры = «Сдаться» (завершает забег с наградой).

## 5. Авторизация и облачный прогресс (PR #14)

- `server/`: Express + `node:sqlite` (БД `server/data/td.db`, gitignored). bcrypt + JWT.
- Эндпоинты: `POST /api/register`, `POST /api/login`, `GET|PUT /api/progress`.
- Фронт: `src/ui/auth.ts` (токен в localStorage), экран «Аккаунт», `storage.loadMetaAsync`. Vite proxy `/api`→`:3000`.
- **Грабли:** `node:sqlite` экспериментальный (`--experimental-sqlite`); prepared statements финализируются → готовить inline, не кэшировать; `server/` не в tsconfig.
- **Прод:** на GitHub Pages сервера нет → только гость. Для auth на проде — захостить сервер отдельно (Render/VPS), `JWT_SECRET` в env. **Не сделано.**

## 6. Ключевые дизайн-решения и «почему» (не потерять)

- **Урон базе:** «прошёл — исчез» (`lapsDamage:false`). Радиусы атаки НЕ добавлять (все бьют весь трек). — DECISIONS.md §14.
- **Правило мерджа:** мердж-по-рангу (PR #5) владельцу показался слишком казуальным → откатили. Строгий тип+ранг + умный призыв (PR #6). Архив в ветке `merge-by-rank`.
- **DoT — %ТЕКУЩЕГО hp, не max:** %maxHP давал имбу (убивает за фикс-время независимо от HP, ломает scaling — колоды улетали за 600 волну). %current затухает, сам не убивает.
- **Заморозка → иммунитет 6с:** иначе Ливень+Мороз = вечный стан. Общий `tryStun` для заморозки и Медузы.
- **Разряд не разносит Мокро/Заряд:** иначе лавина мокрых целей.
- **Support-башни НЕ вводим:** держать башню ради ауры скучно; усиление статусов ушло в апгрейд.
- **HP-кривая гладкая** `(1+0.1X)·1.05^X` вместо ступенек (была стена ×1.67/волну на 30-35).
- **Крит от прокачки классов** — как в Random Dice.

## 7. Балансные находки (это игра ботов, не людей — оговорка)

- Статусные/комбо-колоды сильнее дефолта (медиана ~57-65 vs ~37), но не имба (p75 конечны). Архетип-колоды 34-54.
- Генератор НЕ доминирует (36% маны, +8% к волне). Снайпер overkill 18%→6% после фикса таргетинга (по текущему hp).
- Инструменты: `balance.ts` (`npm run balance N`), scratchpad `elements.ts` (18 тестов реакций), `deadlock.ts`, `combo.ts`.
