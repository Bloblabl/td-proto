# TD-Proto — Башни и синергии: предложения

Дизайн-документ · 22.07.2026 · черновик для обсуждения

Цель: расширить набор башен с 7 до ~20, но не «ради количества», а чтобы каждая
открывала тактику. Ядро предложения — **система элементов-статусов в духе Magicka**,
где статусы на монстрах реагируют друг с другом, и башни ценны не только своим
уроном, но и тем, какие реакции они запускают. Идея с Разрядом, разносящим эффекты
по цепи, встроена в эту систему как одна из ключевых механик.

---

## 1. Что берём у референсов

### Random Dice (100 кубиков, 4 редкости, классы attack / auxiliary attack / auxiliary)

Важное наблюдение: у Random Dice **крит растёт от прокачки классов кубиков** — ровно
то, что мы уже сделали в мете (уровень любой башни → общий крит). Значит направление
верное. Полезные паттерны механик:

| Кубик (референс) | Механика | Что берём |
|---|---|---|
| Fire / Flame / Overheat | Поджигает, DoT горением | Статус «Горение» |
| Poison / Infect / **Metastasis** | Яд DoT; Infect/Metastasis **распространяет яд на соседей** | Разнос статусов (ваша идея!) |
| Electric / Lightning Cloud / Storm | Цепь молний по нескольким целям | Уже есть (Разряд) |
| Ice / Blizzard / Snowball | Заморозка, замедление зоной | Уже есть (Мороз) |
| Wind / Mighty Wind | Скорость атаки растёт с каждым выстрелом | Башня «Вихрь» |
| Growth / Random Growth | Урон растёт со временем, слабый старт | Башня «Ростовик» |
| Critical | Даёт крит-бонус **соседним** кубикам | Башня-аура «Жрец» |
| Mimic / Clone / Joker | Копирует тип соседа | Башня «Мим» |
| Death / Assassin / Predator | Шанс мгновенно убить / добить слабых | Башня «Палач» |
| Nuclear / Atomic | Ядерный взрыв, большой AoE | Буст «Метеор» (есть) / «Пушка» |
| Crack / Thorn / Earthquake | Снимает броню, уязвимость | Статус «Разлом» |
| Medusa | Окаменение (стан) | Башня «Медуза» |
| Wave / Tsunami / Flow / Bubble | Вода | Статус «Мокро» (enabler) |
| Healing / Shield / Barrier | Поддержка, защита | (в TD-Proto база одна — заменяем на баф-ауры) |

### Rush Royale (та же механика merge-by-rank; юниты делятся на damage / support / control)

Ключевой вывод из тир-листов: **сильный состав = дамагер + минимум один support +
один control**. У нас сейчас перекос в дамагеров, почти нет support и control-глубины.

| Юнит (референс) | Роль | Механика | Что берём |
|---|---|---|---|
| Sharpshooter | damage | Одиночный по дальней/жирной цели | Снайпер (есть) |
| Dryad | damage | Урон по площади, **+10% за каждую атаку** (растёт), яд | Ростовик + Яд |
| Inquisitor | control/dmg | Урон растёт с атаками; масштаб от количества копий | Вихрь / Разряд |
| Chemist | support | Усиливает силу ядов и статусов | → перенесено в **апгрейд** (%HP-DoT), не башня |
| Priestess / Engineer / Bard | support | Лечат/бафают/чинят соседние башни | **Не берём** (лечить некого, ауры скучны) |
| Harlequin | utility | Мимик, сглаживает неудачные мерджи | Башня «Мим» |
| Corsair / Cannoneer | damage | Бомбы, тяжёлый AoE | Башня «Пушка» |
| Executioner / Demon Hunter | damage | Добивание, бонус по элите | Башня «Палач» |
| Trapper | control | Ловушки на пути | Наши заграждения (есть) |

**Вывод:** берём не отдельные способности, а две вещи — (1) статусы, которые
**реагируют** между собой (ядро — Magicka-модель); (2) несколько ярких
**damage-архетипов** (растущий, добивающий, разгонный, тяжёлый AoE). Роль support
из Rush Royale сознательно **не берём** — усиление уходит в апгрейд, а не в
отдельную башню.

## 2. Модель Magicka: элементы как статусы + реакции

Суть, которая вам нравится в Magicka: элементы не просто «стакаются», а **вступают в
реакции**. Переносим это на монстров: каждая башня накладывает **статус-элемент**, а
некоторые сочетания дают **реакцию** — разовый мощный эффект.

### Статусы (висят на монстре)

| Статус | Элемент | Базовый эффект | Кто накладывает |
|---|---|---|---|
| **Мороз** | Cold | Замедление + **уязвимость** (+30% урона, уже в игре) | Мороз, Слоу-зона |
| **Яд** | Poison | DoT (гибрид, см. ниже); вытесняется Горением | Яд |
| **Горение** | Fire | DoT (гибрид); «топливо» для детонации; вытесняет Яд, гасится Мокрым | Пиромант |
| **Мокро** | Water | Сам по себе слаб; **enabler** для реакций. **Не разносится и не копируется** | Ливень |
| **Заряд** | Lightning | Цель «наэлектризована»; Разряд по ней бьёт цепью дальше. **Не разносится** | Разряд |
| **Разлом** | Earth | Снята броня: +урон физикой, медленнее движется | Дробитель |
| **Метка** | Arcane | Детонатор: следующая реакция взрывает все статусы разом | Арканист |

### Модель DoT (яд, горение) — гибрид, масштабируется от уровня

Плоский DoT бесполезен в поздней игре: 40/с — ничто против моба на 100k HP.
Поэтому DoT = **плоская часть** (хороша против стай) + **процентная часть от
max HP цели** (актуальна против жирных). Процентная часть **растёт от in-match
уровня башни** — то есть апгрейд Яда/Горения даёт «+N% HP/с к эффекту», а не только
+% к плоскому урону.

```
dotPerSec = base·effMult  +  pctPerLevel·(уровень−1)·targetMaxHP·bossCap
```

- `base·effMult` — плоская часть, как сейчас (уровень типа + драфт + мета).
- `pctPerLevel·(уровень−1)` — процент от max HP цели за каждый уровень апгрейда.
- `bossCap` — понижающий множитель для боссов (как `slowCap`), чтобы не таяли
  мгновенно. Точные числа — через `npm run balance`.

Это заодно **заменяет отдельную башню-усилитель статусов** (Алхимик убран):
усиление ядов живёт в апгрейде, а не в лишней башне.

### Таблица реакций (ядро системы)

| Комбо | Реакция | Эффект |
|---|---|---|
| **Мокро + Мороз** | Заморозка | Полный **стан** на 1–2 с вместо слоу (Magicka: вода замерзает мгновенно) |
| **Мокро + Разряд** | Проводимость | Цепь бьёт **мокрых в радиусе прыжка**, +урон (см. защиту от лавины) |
| **Мокро + Горение** | Пар | Гасит горение, лёгкий AoE, снимает Мокро (баланс: вода тушит) |
| **Яд + Горение** | Детонация | Взрыв AoE, тратит оба статуса (химический ожог) |
| **Разряд по статусной цели** | Разнос | Цепь **копирует Яд / Горение / Мороз** цели на звенья ← **ваша идея** |
| **Разлом + любой физ-удар** | Пробой | Крит-подобный множитель по цели без брони |
| **Метка + N статусов** | Аркан-взрыв | Детонирует **все** статусы на цели одновременно, урон ×(число статусов) |

Правила в духе Magicka: **противоположности вытесняют** (Горение вытесняет Яд;
Мокро/Мороз гасят Горение). Новый статус замещает конфликтный и запускает реакцию.
Решение — не «лей всё подряд», а «в каком порядке накладывать».

### Защита от лавины (ответ на вопрос про имбу молнии)

Если бы Разряд разносил **Мокро**, возникла бы самоусиливающаяся петля: намочил
одного → цепь копирует Мокро → мокрых больше → следующая молния бьёт всех и мочит
ещё дальше → лавина, которую нельзя сбалансировать числами. Поэтому:

- **Мокро и Заряд НЕ разносятся и не копируются.** Разнос берёт только «страдательные»
  статусы — Яд, Горение, Мороз.
- **Мокро накладывает только Ливень.** Чтобы цепь била широко, надо потратить слот и
  прицел Ливня на площадь — это действие с ценой, а не бесплатное распространение.
- **Проводимость ограничена** радиусом прыжка `chainRadius` и числом звеньев
  `chainCount` — «мокрые рядом», а не «все мокрые на карте».

Так молния остаётся сильной в комбо, но не плодит себе цели бесконечно.

## 3. Предложенный список башен

Роли: **DPS** (чистый урон), **Статус** (накладывают элементы), **Специальные**
(уникальная механика). Support-башни (Жрец, Алхимик) **убраны** по решению владельца —
держать башню только ради ауры скучно, а лечить в TD-Proto некого. Усиление статусов
перенесено в апгрейд (см. модель DoT). Характеристики — ориентир в формате нашего
`balance.json` (точные числа — через `npm run balance`).

## 4. Матрица синергий и комбо (именованные тактики)

Именованные комбо — это то, что игрок открывает и запоминает; они и создают
«ещё один заход».

| Тактика | Состав | Что происходит |
|---|---|---|
| **«Эпидемия»** | Яд + Разряд | Разряд бьёт отравленную цель → яд разносится по цепи (ваша идея в чистом виде) |
| **«Проводник»** | Ливень + Разряд | Мокрые цели → цепь бьёт мокрых в радиусе, +урон (Мокро не копируется — без лавины) |
| **«Криоказнь»** | Ливень + Мороз + Снайпер | Мокро+Мороз = стан, Снайпер бьёт обездвиженную жирную цель |
| **«Химический ожог»** | Яд + Пиромант | Детонация: взрыв AoE на стеке яда+горения |
| **«Аркан-бомба»** | Мороз + Яд + Пиромант + Арканист | Копим 3 статуса, Метка детонирует всё разом, урон ×(число статусов) |
| **«Бронебойня»** | Дробитель + Стрелок/Снайпер | Разлом снимает броню → пробой, добивание |
| **«Фокус боссу»** | Вихрь + Мороз | Заморозка держит босса, разогнанный Вихрь копит скорость burst'а |
| **«Ковёр»** | Мороз + Пушка/Залп | Заморозка группирует стаю → тяжёлый AoE накрывает всех |
| **«Токсиколог»** | прокачанный Яд + Пиромант | Апгрейд даёт %HP-DoT → детонация жирных целей кратно сильнее |

**Общий принцип (Magicka):** статусные башни сами по себе слабы по урону (как Мороз
сейчас — 5%), но в **правильном порядке и сочетании** дают кратный выхлоп. Это
смещает решение с «поставь больше пушек» на «собери работающую комбинацию» — именно
та глубина, которой сейчас не хватает.

## 5. Реализуемость в текущем ядре

Ядро (`src/core/sim.ts`, `types.ts`) уже готово к большей части — статусы и
уязвимость есть.

**Малой кровью (данные + немного логики):**
- Новые статусы: добавить в `Monster` поля `burnUntil`, `wetUntil`, `chargedUntil`,
  `markUntil`, `sunderUntil` — по образцу существующих `slowUntil`, `dotUntil`.
- Реакции: единая функция `applyStatus()`, проверяющая комбо и вытеснение
  (Горение ↔ Мокро). Место есть — сейчас статусы ставятся прямо в цикле атак.
- Разнос статусов Разрядом: в существующей chain-логике копировать статусы цели на
  следующее звено. ~10 строк.
- Обобщить `slowVulnMult` в набор множителей от статусов (Разлом, Метка).
- Новые DPS-башни на существующих `targeting` (first/maxhp): Пиромант, Ливень,
  Пушка, Палач, Вампир — конфиг + маленькие правки.

**Требует расширения ядра:**
- **Поддержка-ауры** (Жрец, Алхимик): новый режим `targeting: 'aura'` + расчёт
  бонусов соседям по клеткам поля. Средняя работа.
- **Вихрь** (растущая скорость по цели), **Ростовик** (рост по волнам): поля
  состояния на юните. Малая работа.
- **Мим** (копия соседа): логика при установке/мердже. Средняя.
- **Медуза** (стан): новый статус «обездвижен» (движение = 0). Малая.

**UI/рендер:** каждому статусу — свой цвет обводки/иконка на монстре (сейчас есть
голубой=слоу, зелёный=яд). Реакции — вспышка. Умеренная работа.

## 6. Приоритеты внедрения

Чтобы система элементов заиграла как можно раньше, вводим не по одной башне, а
**связками, которые сразу дают реакции**.

**Волна 1 — «зажечь систему» (раскрывает Magicka-ядро, минимум нового кода):**
- Башни: **Пиромант** (Горение), **Ливень** (Мокро).
- Механики: разнос статусов Разрядом, реакции Детонация / Проводимость / Заморозка.
- Результат: сразу играбельны «Эпидемия», «Проводник», «Химический ожог»,
  «Криоказнь». Мороз и Яд из «слабых пушек» становятся частью комбо.

**Волна 2 — «глубина статусов»:**
- Гибридный DoT (%HP от уровня башни) → апгрейд усиливает эффекты.
- Башни: **Дробитель** (Разлом), **Арканист** (Метка/детонатор), **Медуза** (стан).
- Результат: «Аркан-бомба», «Бронебойня», «Токсиколог» — комбо на детонацию и контроль.

**Волна 3 — «архетипы и глубина»:**
- Башни: **Вихрь**, **Пушка**, **Ростовик**, **Мим**, **Палач**, **Вампир**.
- Редкости карточек драфта (common/rare/legendary) под новый пул.

Каждая волна — отдельный PR с проверкой баланса `npm run balance` и smoke-гейтом,
как делали раньше.

## Решённые вопросы (владелец, 22.07.2026)

1. **Колода — оставляем 5 из 7**, не расширяем. Значит комбо должны работать на
   компактных составах: реакции — на парах/тройках башен, не на «соберись из 12».
2. **Статусы — по канону Magicka: вытеснение + реакции**, не бесконечный стак.
   Противоположности замещают друг друга (Горение↔Мокро/Мороз).
3. **Support-башни убраны.** Держать башню только ради ауры скучно, лечить некого.
   Усиление статусов ушло в апгрейд (гибридный %HP-DoT).
4. **Молния не разносит Мокро/Заряд** — только Яд/Горение/Мороз (защита от лавины).
5. **DoT гибридный** — плоская часть + %HP от уровня башни, с капом на боссов.
